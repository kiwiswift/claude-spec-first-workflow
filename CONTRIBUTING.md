# Contributing to Specification-First Workflow

Thank you for your interest in improving the Specification-First Workflow!

## How to Contribute

### Reporting Issues

If you find a bug or have a suggestion:

1. Check existing issues first
2. Create a new issue with:
   - Clear title and description
   - Steps to reproduce (for bugs)
   - Expected vs. actual behavior
   - Your Claude Code version
   - Your project type (TypeScript/Python/etc.)

### Suggesting Improvements

We welcome suggestions for:
- New slash commands
- Template improvements
- Documentation enhancements
- Auto-detection logic
- Quality checks

### Submitting Changes

1. **Fork the repository**

2. **Create a feature branch**
   ```bash
   git checkout -b feature/amazing-improvement
   ```

3. **Make your changes**
   - Update documentation if needed
   - Follow existing code style
   - Test your changes

4. **Commit with clear messages**
   ```bash
   git commit -m "feat: add dependency graph visualization"
   ```

5. **Push and create PR**
   ```bash
   git push origin feature/amazing-improvement
   ```

## Development Guidelines

### For Command Changes

- Update the command markdown file
- Test with real project scenarios
- Update README if behavior changes
- Add examples to documentation

### For Template Changes

- Ensure backward compatibility
- Provide clear examples
- Update template documentation
- Consider different project types

### For Steering Document Changes

- Keep language-agnostic when possible
- Provide templates for common stacks
- Document detection logic
- Test auto-creation

## Testing Your Changes

Before submitting:

1. Test with a real project
2. Try all affected commands
3. Verify documentation is accurate
4. Check that examples work

## Documentation Standards

- Use clear, concise language
- Provide examples
- Include code snippets
- Keep formatting consistent

## Code of Conduct

- Be respectful and inclusive
- Focus on constructive feedback
- Help others learn

## Questions?

Feel free to:
- Create a discussion
- Ask in issues
- Reach out to maintainers
